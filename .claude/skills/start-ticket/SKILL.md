# チケット開発開始

## 実行指示

引数で指定されたチケット ID（例: `PROJ-123`）をもとに、開発を始める準備をすべて行ってください。

チケット ID が指定されていない場合は「チケット ID を引数で指定してください（例: /start-ticket PROJ-123）」と返してください。

---

### ステップ 1: Jira チケットの取得・分析

Jira MCP でチケットを取得し、以下を把握する：

- タイトル・説明・受け入れ条件
- 実装に関わるコンポーネント・技術スタック（記載があれば）
- 関連チケット（あれば）

---

### ステップ 2: Kibela で関連情報を調査

チケットの内容から **2〜3 個のキーワード** を抽出して Kibela を検索し、関連する仕様書・設計ドキュメント・過去の実装記録を取得する。

見つかった記事の要点を箇条書きでまとめる。

---

### ステップ 3: ブランチ作成

- ブランチ名を生成する（形式: `{チケット ID}/{タイトル要約}`、例: `VOC-808/add-login-validation`）
- ユーザーに提示して確認後、`git checkout -b` で作成する

---

### ステップ 4: 実装のたたきを提示

チケットと Kibela の情報をもとに、以下を提示する：

```
## 実装方針

### アプローチ
（どう実装するかの方針を 3〜5 行で）

### 変更が必要なファイル（推定）
- `path/to/file.ts` — （何をするか）
- ...

### 実装ステップ
1. （最初にやること）
2. （次にやること）
3. ...

### 注意点・考慮事項
- （Kibela や仕様から読み取った注意点）
- ...
```

---

### 最後に

「実装を始めますか？それとも方針について確認しますか？」と聞く。
