## 日報システム (Daily Report System)

### 初期設定
```bash
chmod +x scripts/nippo_append.sh
```

### 時短作業
日報を開く
```
<leader>no
```

作業を追記
```
<leader>na
```

終業時に日報を完成させる
```
<leader>nc
```

```
:cc
```

生成された完成版を日報ファイルに反映

### 基本的な使い方
```bash
# 基本的な作業ログ追記
npa 今日の作業内容を記録

# セクション別の追記
npa 目標:API開発を完了させる       # 🎯 今日の目標
npa 進捗:設計が80%完了           # 📊 進捗状況  
npa 学び:新しいライブラリを習得    # 💡 学びと気づき
npa 気づき:テストが重要          # 💡 学びと気づき
npa 振り返り:効率的に進められた   # 💡 学びと気づき
npa 明日:テストコードを作成      # 🚀 明日への申し送り
npa 目標達成:API開発            # 該当目標をチェック完了
```

### 複数項目の組み合わせ
```bash
# 1つのコマンドで複数のセクションに追記可能
npa 目標:データベース設計 進捗:50%完了 学び:ORMの使い方 明日:テスト作成

# 同じセクションに複数項目も可能
npa 学び:Docker基礎 学び:CI/CD 気づき:自動化が重要 気づき:コードレビューが有効
```

### 生成される日報の構造
```markdown
# 日報 2026年01月14日

## 📝 作業ログ
### 22:51 - 作業内容
作業内容

## 🎯 今日の目標
- [ ] （後で記入）
- [ ] API開発を完了させる

## 📊 進捗状況
（セッション終了時に記入）
• 設計が80%完了

## 💡 学びと気づき
（随時追記）
• 新しいライブラリを習得
• テストが重要

## 🚀 明日への申し送り
（本日終了時に記入）
• テストコードを作成
```

### ファイル保存場所
- 日報ファイル：`~/dotfiles/daily_reports/daily_report.YYYY-MM-DD.md`
- 自動的にディレクトリが作成されます
